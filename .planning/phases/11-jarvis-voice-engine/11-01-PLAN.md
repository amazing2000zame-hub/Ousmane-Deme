# 11-01: TTS Provider + Backend Endpoint + Voice Store

## Goal
Backend TTS service using OpenAI TTS API with REST endpoint, plus frontend Zustand voice store with localStorage persistence.

## Files to Create
- `jarvis-backend/src/ai/tts.ts` — TTS provider interface with OpenAI implementation
- `jarvis-backend/src/api/tts.ts` — POST /api/tts endpoint returning audio/mpeg stream
- `jarvis-ui/src/stores/voice.ts` — Zustand voice state (enabled, autoPlay, speed, voice)

## Files to Modify
- `jarvis-backend/src/config.ts` — Add TTS config vars
- `jarvis-backend/src/index.ts` — Mount TTS router
- `jarvis-backend/.env.example` — Add TTS vars

## Design
- OpenAI TTS API (`tts-1` model) with "onyx" voice (deep, authoritative JARVIS)
- Backend streams audio response directly to client (no intermediate file)
- Voice store persists settings to localStorage
- Provider-agnostic: supports 'openai' and 'browser' (Web Speech API fallback)
